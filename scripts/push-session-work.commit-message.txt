D-016: compliance gate moves from bid-submit to bid-award

Backend (functions/src):
- bids/awardBidRoute.ts (NEW): POST /api/bids/award. Staff-only. Resolves
  the bid -> winning sub's users/{uid} doc (tries submittedByUid, then
  subContactId.linkedUserId, then legacy subId). Checks W-9 + insurance
  + signed agreement + contractor license number. Returns 400 with
  { error, missingItems: [...] } if any are missing — the GC modal
  surfaces this as a destructive toast naming each missing item.
  On success: atomic batch flips bid.status + bidRequest.status to
  'awarded', notifies the winning sub.
- index.ts: registered the award route before the catch-all 404.

Client (client/src):
- pages/BidRespond.tsx: verification WALL removed. Response CTA renders
  unconditionally for any signed-in sub. Advisory compliance card below
  shows the 4-item checklist with inline text input for contractor
  license number — writes straight to users/{uid}.contractorLicenseNumber
  via the existing user-self-write Firestore rule. No more "Complete
  verification" button (which pointed to /sub/onboarding — a route that
  never existed and 404'd users out of the flow).
- components/bidding/SubBidSubmissionForm.tsx: complianceComplete check
  removed from handleSubmit. Submit button no longer disabled on missing
  compliance. Banner restyled amber + reworded ("Verification needed
  before Skyeline can award this work"). Fourth item (contractor license)
  added to the compliance check.
- components/bidding/AwardBidModal.tsx: markBidAwarded() now POSTs to
  /api/bids/award with a bearer token instead of doing client-side
  Firestore writes. Server-side gate handles the rest; the server's
  error string (with explicit missingItems) is passed through to the
  user as the toast description.
- components/bidding/SendBidPackageModal.tsx: modal title "Send Bid
  Package" -> "Create new bid package" (consistent with the entry
  button). Submit button "Send package (N subs)" -> "Send to N subs"
  (the create happens before the modal opens; the action inside the
  modal is the actual send).

Docs:
- docs/decisions.md: D-016 added with full rationale. Why this is better
  than the old gate (subs aren't dead-ended, Skyeline still controls
  risk, kills the 404 from the broken /sub/onboarding link). Where
  each compliance item is captured. Three sub-decisions (D-016-a/b/c)
  for the upload UI, expiration tracking, and notification target
  follow-ups.

Auto-deploy:
- scripts/auto-push.sh (NEW): Claude can now push from the sandbox
  using the PAT at .claude-push-token (gitignored). Fresh /tmp clone +
  rsync from workspace + commit + push, with the credential masked in
